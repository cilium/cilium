# Changelog

## v1.20.0-pre.0

Summary of Changes
------------------

**Major Changes:**
* Add support for Kubernetes Cluster Network Policy (KCNP) (cilium/cilium#42338, @TheBeeZee)
* Gateway API support now includes BackendTLSPolicy features (cilium/cilium#43045, @youngnick)

**Minor Changes:**
* [41867] Part 3: Add subnet config watcher with stateDB and BPF map sync (cilium/cilium#43438, @anubhabMajumdar)
* [CFP-39876]: Add namespace filtering conditions to ServiceImport controller (cilium/cilium#44040, @jimassa)
* Add detection of unknown keys in the cilium-config ConfigMap during preflight for the agent and operator. (cilium/cilium#43025, @andy176631)
* Add option to create CiliumEndpointSlices directly from pods instead of CiliumEndpoints (cilium/cilium#38388, @jshr-w)
* Add support for wildcard specifier anywhere in SNI pattern (cilium/cilium#43674, @fristonio)
* bpf: lxc: also handle non-DSR Nodeport services in per-packet LB path (cilium/cilium#44507, @julianwiedmann)
* cli: clustermesh: use ca bundle to connect clusters (cilium/cilium#42833, @MrFreezeex)
* clustermesh: remove components restart when providing directly IP (no domain) to connect to remote clustermesh-apiserver (cilium/cilium#44425, @MrFreezeex)
* Explicitly set `hostUsers: true` on containers where user namespaces would break the runtime. (cilium/cilium#43615, @jcpunk)
* Generalize the node-port nat conflict detection by removing dependency on direct routing interface (cilium/cilium#43955, @ldelossa)
* Improve probing for necessary kernel functionality required for service backend termination to avoid false positives. (cilium/cilium#42867, @tommyp1ckles)
* ingress: allow per-protocol port overrides for HTTP, HTTPS and TLS passthrough in the HostNetwork mode. (cilium/cilium#44447, @viktor-kurchenko)
* Introduces "auto" datapath-mode. If set, Cilium will probe the underlying host for netkit device support at startup. If supported, pods will be created with netkit devices, otherwise veth pairs will continue to be used. (cilium/cilium#43062, @ajmmm)
* iptables:  Allow Cilium to start in environments where `ip6tables` is not available but iptables rule installation is disabled via configuration. (cilium/cilium#43940, @javiercardona-work)
* k8s: remove permissions for list/get/watch of endpoints (cilium/cilium#42760, @marseel)
* Metrics are collected for SCTP connections. (cilium/cilium#43535, @Jack-R-lantern)
* metrics: remove agent bootstrap metrics (cilium/cilium#44180, @mhofstetter)
* Provide a new annotation that enables dynamic pod routing. (cilium/cilium#44319, @ldelossa)
* Remove deprecated "aws-pagination-enabled" option in Cilium operator in favor of "aws-max-results-per-call" (cilium/cilium#43693, @pippolo84)
* Removed support for Envoy Go Extensions (proxylib) and Kafka-aware network policies. These features were deprecated in v1.16. Users relying on Kafka L7 policy enforcement should migrate to CiliumEnvoyConfig-based solutions. (cilium/cilium#43557, @sayboras)
* Removed the `encryption.ipsec.interface` Helm value and its associated `bpf_network.c` program, which had been unused since at least Cilium 1.18. (cilium/cilium#44284, @ti-mo)
* The dummy endpoint (192.192.192.192:9999) is no longer created for Ingress and Gateway API. (cilium/cilium#43558, @joamaki)
* The previously deprecated Helm value `clustermesh.enableMCSAPISupport` was removed in favor of the `clustermesh.mcsapi.enabled` Helm value (cilium/cilium#44300, @MrFreezeex)
* ztunnel/helm: move ztunnel daemonset management from operator to helm (cilium/cilium#43763, @nddq)

**Bugfixes:**
* agent:overlay: Add underlayProtocol 'Auto' for automatic underlay selection (cilium/cilium#43057, @smagnani96)
* Azure IPAM: Optimize subnet discovery to eliminate Azure NRP throttling by using targeted subnet queries instead of subscription-wide VNet enumeration, significantly improving performance in large Azure environments with many VNets. (cilium/cilium#41555, @yuecong)
* Fix a bug where node IPv6 updates and deletes were not correctly propagated to the Linux kernel neighbor subsystem. (cilium/cilium#44540, @tklauser)
* Fix envoy admin socket being created as world-accessible (cilium/cilium#44512, @0xch4z)
* Fix in-cluster NodePort connectivity failure in DSR mode when SocketLB is disabled. When a pod accesses a NodePort service via a remote node's IP (instead of the ClusterIP) and the selected backend resides on the same node as the client, the connection fails due to missing reverse NAT on the reply path. (cilium/cilium#41963, @gyutaeb)
* Fix: Cilium Ingress now automatically reallocates ports and retries when cilium-envoy fails to bind due to port conflicts (cilium/cilium#42859, @inerplat)
* Fixes increased CPU usage in `hubble observe` caused by log coloring feature, even when coloring was disabled (cilium/cilium#44119, @tporeba)
* generate the proper logs when users put 0.0.0.0/0 as the native routing range for iptable nft mode (cilium/cilium#43415, @liyihuang)
* helm/ztunnel: Add host field to readiness probe to bind the health check port 15021 to 127.0.0.1 instead of 0.0.0.0 (cilium/cilium#44196, @nddq)
