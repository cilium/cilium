# Changelog

## v1.18.0-pre.2

Summary of Changes
------------------

**Major Changes:**
* Implement the same level of support for fragmented IPv6 fragments as we have for IPv4. (cilium/cilium#38110, @gentoo-root)
* ipsec: implement VinE traffic (cilium/cilium#37723, @ldelossa)
* Promote `CiliumLoadBalancerIPPool` CRD to v2 API version. (cilium/cilium#39090, @pippolo84)
* Support IPv6 as a tunneling underlay. (cilium/cilium#38296, @pchaigno)
* Support KPR with IPv6 underlay (cilium/cilium#39074, @pchaigno)

**Minor Changes:**
* Add `cilium shell -- health` command that takes a optional prefix and prints the reporter tree similar to current `cilium status --verbose` output. (cilium/cilium#38076, @tommyp1ckles)
* Add `cilium-dbg bpf ipcache match` command to lookup ipcache entries using an exact match (cilium/cilium#38579, @pippolo84)
* Add serviceaccount label in the default labels list (cilium/cilium#38017, @liyihuang)
* Add support for IPv6 egress gateway policies (cilium/cilium#38452, @rgo3)
* agent: deprecate --bpf-lb-proto-diff (cilium/cilium#39259, @julianwiedmann)
* AWS ENI IPAM: disable internal `aws-sdk-go-v2` client rate-limiter which interfered with Cilium's rate-limiter (cilium/cilium#38550, @antonipp)
* bandwidth: Introduce bbrHostNamespaceOnly to allow limited use of BBR in legacy routing mode. (cilium/cilium#38898, @jrife)
* bgp: Ensure reconciliation of services with externalTrafficPolicy=Local upon endpoint/endpointslice deletion. (cilium/cilium#38966, @rastislavs)
* bugtool:feature: accept pattern for dumping matching pinned maps (cilium/cilium#38690, @smagnani96)
* Cilium Agent liveness probe no longer fails if Kubernetes apiserver cannot be reached. Earlier the agent was restarted if the apiserver could not be reached for approximately 5 minutes. This avoids traffic disruptions on apiserver downtime (e.g. due to maintenance) for features such as L7 and FQDN proxy that require cilium-agent to always be up. (cilium/cilium#38458, @joamaki)
* Cilium's CNI plugin now conforms to CNI v1.1 and reports per-route MTUs. (cilium/cilium#38868, @squeed)
* cilium-cli: Fix ipv6 ping regex (cilium/cilium#38814, @yrsuthari)
* feat: Support IPv6 for delegated IPAM (cilium/cilium#38249, @kadevu)
* helm: only expose the envoy admin debug port for cilium-agent when it is explicitly enabled (cilium/cilium#39194, @becker-s)
* hubble: send server version using metadata in gRPC responses (cilium/cilium#38668, @devodev)
* Introduce `--underlay-protocol` (cilium/cilium#38523, @pchaigno)
* k8s/labels: Refactor `FindReserved` to return `LabelArray` (cilium/cilium#39144, @doniacld)
* k8s: Promote CCG to v2 and deprecate v2alpha1 (cilium/cilium#38940, @christarazi)
* policy: Add validation and docs for TLS SNI ServerNames (cilium/cilium#38615, @sayboras)
* Remove deprecated CONNTRACK_LOCAL (cilium/cilium#38687, @tklauser)
* Remove insertions and deletions to deprecated tunnel map (cilium/cilium#38490, @pippolo84)
* Since pod CIDRs are now stored into the ipcache map, tunnel map is not needed anymore. Any reference to the tunnel map have been removed from cilium-dbg, cilium status and bugtool. (cilium/cilium#38839, @pippolo84)
* The Grafana dashboard now displays policy drops in both directions. (cilium/cilium#37445, @squeed)
* The service load-balancing control-plane in the Cilium agent has been redesigned which reduces memory usage and improves future extendability of load-balancing features. (cilium/cilium#38469, @joamaki)
* Update k8s tests and libraries to v1.33.0 (cilium/cilium#39124, @sayboras)
* Update k8s tests and libraries to v1.33.0-rc.1 (cilium/cilium#39080, @sayboras)
* Update kafka apiKey helm chart value to true (cilium/cilium#38963, @kyle-c-simmons)

* An option was added to control SO_LINGER config on Envoy HTTP upstream connections that bind to the pod's original source address and port. This can be used to mitigate bind errors that could happen if the upstream HTTP connection can not be cleanly closed immediately. (cilium/cilium#38500, @jrajahalme)
* bgpv2: Fix service reconciliation by BGP peer IP change (cilium/cilium#38620, @rastislavs)

* Fix a deadlock when a host has no IPv4 address. (cilium/cilium#38938, @EmilyShepherd)
* Fix a panic happening in the ipset reconciler when a previous reconciliation failed. (cilium/cilium#38890, @pippolo84)
