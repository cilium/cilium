# Changelog

## v1.18.0

Summary of Changes
------------------

**Major Changes:**
* Add support for IPv6 egress gateway policies (cilium/cilium#38452, @rgo3)
* Add support for kube-apiserver high availability with kube-proxy replacement where the Cilium agent can fail over to an active kube-apiserver at runtime. (cilium/cilium#37601, @aditighag)
* Add support for kube-proxy replacement with IPv6 underlay (cilium/cilium#39074, @pchaigno)
* Add support for multiple gateways to Cilium Egress Gateway. (cilium/cilium#39304, @carlos-abad)
* Add support for ordered IPv6 fragments (cilium/cilium#38110, @gentoo-root)
* Add support for VXLAN in IPsec (VinE) (cilium/cilium#37723, @ldelossa)
* Added an experimental xDS client library (cilium/cilium#34484, @AwesomePatrol)
* Added multi-device support to the L2 pod announcement feature (cilium/cilium#38198, @dylandreimerink)
* Adds a free-text log field to policies, which is exposed in Hubble flows for easy correlation and searching. (cilium/cilium#39902, @squeed)
* bandwidth: support ingress rate limiting using eBPF token bucket (cilium/cilium#36351, @l1b0k)
* Introduces cilium-config ConfigMap monitoring, allowing the agent to automatically synchronize changes in designated sources with its in-memory database. A new drift checker compares the ingested configurations with the agent's configuration, logging any differences and publishing metrics for easy tracking of configuration drifts. (cilium/cilium#36510, @ovidiutirla)
* Migrate from logrus to slog (cilium/cilium#39664, @aanm)
* Promote `CiliumBGPClusterConfig`, `CiliumBGPPeerConfig`, `CiliumBGPAdvertisement`, `CiliumBGPNodeConfig` and `CiliumBGPNodeConfigOverride` CRDs to v2 API version. (cilium/cilium#37765, @rastislavs)
* Promote `CiliumCIDRGroup` CRD to v2 API version (cilium/cilium#38940, @christarazi)
* Promote `CiliumLoadBalancerIPPool` CRD to v2 API version (cilium/cilium#39090, @pippolo84)
* Support IPsec with IPv6 underlay in IPv6-only clusters (cilium/cilium#39497, @pchaigno)
* Support IPv6 as a tunneling underlay. (cilium/cilium#38296, @pchaigno)
* Support IPv6 for delegated IPAM (cilium/cilium#38249, @kadevu)
* Supporting Gateway API v1.3.0 (cilium/cilium#39590, @sayboras)
* The local REST policy API is deprecated. It will be removed in a subsequent release. (cilium/cilium#40212, @squeed)
* The service load-balancing control-plane in the Cilium agent has been redesigned which reduces memory usage and improves future extensibility of load-balancing features. (cilium/cilium#38469, @joamaki)
