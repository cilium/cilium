# Changelog

## v1.16.0-pre.3

Summary of Changes
------------------

**Major Changes:**
* BGP: New BGP APIs can be used to configure Cilium BGP Control Plane. (cilium/cilium#32426, @harsimran-pabla)
* NAT source port metrics & table (cilium/cilium#32152, @tommyp1ckles)

**Minor Changes:**
* Add `CiliumNodeConfig` CRD on API v2 (cilium/cilium#31721, @doniacld)
* Added a new annotation `ingress.cilium.io/loadbalancer-class` to control the `LoadBalancerClass` of a dedicated LB via the ingress. (cilium/cilium#31650, @Sh4d1)
* cilium-envoy now uses upstream filter chains for L7 LB policy enforcement. (cilium/cilium#32119, @jrajahalme)
* CiliumEnvoyConfig CRDs now support an optional 'ports' field in services objects, limiting the redirected service frontends to the ones whose port is listed. (cilium/cilium#32382, @jrajahalme)
* CiliumNetworkPolicies are now validated by the operator and the result set in the object's Status field. (cilium/cilium#32727, @squeed)
* Do not include the unnecessary "localhost" SAN in autogenerated clustermesh admin certificates (cilium/cilium#32662, @giorio94)
* gateway-api: ALPN support (cilium/cilium#32486, @rauanmayemir)
* Generate SBOMs using Syft instead of bom (cilium/cilium#32307, @ferozsalam)
* Helm: Add new value `.Values.clustermesh.apiserver.tls.enableSecrets. Setting this value to false will disable the creation of TLS certificate secrets for clustermesh, enabling out-of-band TLS certificate secret management. (cilium/cilium#32196, @soggiest)
* Hubble peer's port number is inferred from the agent's configuration instead of assuming defaults (cilium/cilium#32729, @AwesomePatrol)
* hubble: add SNAT IP flow field and filter (cilium/cilium#32130, @kaworu)
* hubble: add support to filter Hubble flow by network interface. (cilium/cilium#32286, @kaworu)
* hubble: add the cluster name to a flow's source and destination endpoints (cilium/cilium#32313, @rolinh)
* Improved background resynchronization of nodes. Before all nodes were being updated at the same time, now we spread updates over time to average out CPU usage. (cilium/cilium#32577, @marseel)
* ingress: request timeout control via operator flag & annotation (cilium/cilium#31693, @a5r0n)
* Introduce CLI commands to troubleshoot connectivity issues to the etcd kvstore and clustermesh control plane (cilium/cilium#32336, @giorio94)
* ipsec: Improve CPU usage of cilum-agent in large clusters (cilium/cilium#32588, @marseel)
* KVStoreMesh: expose remote clusters information and introduce dedicated CLI command (cilium/cilium#32156, @giorio94)
* Make the overwriting behavior of install-plugins.sh configurable. (cilium/cilium#32016, @jingyuanliang)
* Operator: expose remote clusters information through dedicated CLI command, and introduce troubleshoot commands (cilium/cilium#32436, @giorio94)
* pkg/healthv2: reduce unecessary healthv2 debug log volume. (cilium/cilium#32319, @tommyp1ckles)
* Report estimated expiry timers for connection-based FQDN entries (cilium/cilium#32013, @joestringer)
* Runtime device detection and subsequent datapath reconfiguration is now the default and only mode of operation. The enableRuntimeDeviceDetection option is now a no-op and will be removed in v1.17. (cilium/cilium#32153, @joamaki)
* Service connections that use ``Direct-Server-Return`` and were established prior to Cilium v1.13.3 will be disrupted, and need to be re-established. (cilium/cilium#32642, @julianwiedmann)
* Simplify rate limit configuration options for the CiliumEndpointSlice controller. (cilium/cilium#32523, @thorn3r)
* Starting cilium-agent with large numbers of network policies should be much faster. (cilium/cilium#32703, @squeed)
* The StateDB in-memory database library was switched to github.com/cilium/statedb with a new much faster radix tree implementation. This is used internally in the cilium-agent for storing and accessing, among others, the network devices and local node IP addresses. This state can be inspected with the "cilium-dbg statedb" commands. cilium-dbg: Added "statedb ipsets" command cilium-dbg: "statedb sysctl-settings" is now "statedb sysctl" (cilium/cilium#32125, @joamaki)
* Unconditionally require the clustermesh cluster configuration to be always present (cilium/cilium#32505, @giorio94)
