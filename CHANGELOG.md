# Changelog

## v1.18.0-pre.3

Summary of Changes
------------------

**Major Changes:**
* Add support for multiple gateways to Cilium Egress Gateway. (cilium/cilium#39304, @carlos-abad)
* Support IPsec with IPv6 underlay in IPv6-only clusters (cilium/cilium#39497, @pchaigno)

**Minor Changes:**
* Add security context to helm chart (cilium/cilium#39205, @Sindvero)
* Add support for Multi-Pool IPAM mode with IPSec transparent encryption in tunnel routing mode (cilium/cilium#39442, @pippolo84)
* agent: Deprecate --enable-internal-traffic-policy (cilium/cilium#39768, @brb)
* agent: Deprecate enable-hubble-recorder-api and friends (cilium/cilium#39642, @brb)
* BGP: Add peer auto-discovery using default-gateway mode (cilium/cilium#39416, @naveenachyuta)
* clustermesh: add a new MCS-API controller mirroring the local EndpointSlice instead of copying the Service selectors. Cilium MCS-API now support EndpointSlice created outside of Service selectors and it fixes hostname synchronization in the context of MCS-API. (cilium/cilium#38596, @MrFreezeex)
* clustermesh: helm option that allows for disabling KVStoreMesh has been deprecated (cilium/cilium#39785, @marseel)
* Collect bpf-related metrics using native Go code instead of bpftool. Users should see a significant speedup in /metrics scraping as well as a significant decrease in CPU usage related to bpf metrics collection on both large and small Cilium deployments. (cilium/cilium#39557, @ti-mo)
* daemon: Add require-k8s-connectivity flag to enable usage with cloud NLB services (cilium/cilium#39434, @ya-makariy)
* Deprecate --enable-recorder (cilium/cilium#39585, @brb)
* Deprecate enable-{node-port,host-port,externalips} (cilium/cilium#39581, @brb)
* fix: Add support for changing the vxlan tunnel port after initial installation (cilium/cilium#38583, @rapour)
* Helm: Add option to define resource limits for cerngen cronjob (cilium/cilium#39261, @jcpunk)
* helm: KPR subflag changes (cilium/cilium#39721, @brb)
* helm: PodDisruptionBudget can now set unhealthyPodEvictionPolicy (cilium/cilium#39299, @jcpunk)
* hubble-cli: new `--print-policy-names` option to show the names of (C)CNPs that allowed or denied traffic (cilium/cilium#39453, @antonipp)
* images: Bump llvm image to 19.1.7 (cilium/cilium#39632, @sayboras)
* Implemented optional etcd cache for remote cluster's data in `kvstore` identity allocation mode. (cilium/cilium#39523, @balous)
* Improves performance of the DNS proxy when large numbers of IPs are being cached. (cilium/cilium#39340, @squeed)
* Increase the minimum required kernel version to v5.10 / RHEL 8.6. (cilium/cilium#38308, @julianwiedmann)
* Introduce a new ip pool mode for BGP router ID allocation so cilium can pick up the router ID from a pool. (cilium/cilium#38300, @liyihuang)
* pkg/lb: Add service.node.io/node(-selector) support (cilium/cilium#39551, @brb)
* policy: add a new option enable-default-restrict-local-cluster-policy to restrict policies rules to the local cluster in ClusterMesh environment (cilium/cilium#39338, @MrFreezeex)
* proxy: Add deprecated warning for proxylib (cilium/cilium#39419, @sayboras)
* Remove dependency on /boot/config* and/or /proc/config* for feature detection (cilium/cilium#39438, @ti-mo)
* remove deprecated node_connectivity metrics (cilium/cilium#39694, @jshr-w)
* Remove stale ingress and egress routing rules after the deletion of an endpoint. (cilium/cilium#39734, @pippolo84)
* Revert #36978 (cilium/cilium#39333, @brb)
* The deprecated 'enableRuntimeDeviceDetection' helm option has been removed. (cilium/cilium#39633, @joamaki)
* The service upsert and delete notifications are no longer shown as part of "cilium-dbg monitor". Instead the changes to service frontends can be monitored for debugging purposes with "cilium-dbg shell -- db/watch frontends". (cilium/cilium#39493, @joamaki)
* vendor: Bump gateway-api version to 1.3.0 (cilium/cilium#39590, @sayboras)
