# Changelog

## v1.19.0-pre.3

Summary of Changes
------------------

**Major Changes:**
* ztunnel: add xds control-plane (cilium/cilium#42169, @rgo3)
* ztunnel: add ztunnel daemonset controller (cilium/cilium#42766, @ldelossa)

**Minor Changes:**
* Add iptables rule to allow tunnel traffic egressing the node when tunnel routing mode is used (cilium/cilium#42356, @bersoare)
* Add option `externallyCreated` for external creation of service for clustermesh apiserver. (cilium/cilium#42227, @andrejvanderzee)
* Add support for hybrid routing in datapath (cilium/cilium#41868, @anubhabMajumdar)
* bgp: Add SourceInterface configuration option into CiliumBGPTransport (cilium/cilium#42583, @rastislavs)
* bgp: Introduce "Interface" BGP advertisement type allowing advertisement of arbitrary IPs assigned on local interfaces (cilium/cilium#42469, @rastislavs)
* Change the sidecar etcd instance of the Cluster Mesh API Server listen on all IP addresses (cilium/cilium#42818, @giorio94)
* clustermesh: add MCS-API CRDs auto-installation (cilium/cilium#40729, @MrFreezeex)
* clustermesh: deprecate passing tls cert/keys in Helm values (cilium/cilium#42576, @MrFreezeex)
* clustermesh: set tls authMode to migration by default (cilium/cilium#42866, @MrFreezeex)
* Delays CiliumIdentity allocation for a short random period of time, which can help reduce duplicate identity creation. (cilium/cilium#42282, @squeed)
* Enable PLPMTUD on endpoint netns (cilium/cilium#42012, @tommyp1ckles)
* helm: Use named ports for container probes (cilium/cilium#39281, @jcpunk)
* Hubble v1.Events drop message now supports displaying applied L4 network policies (cilium/cilium#41693, @41ks)
* ingress: Replace Endpoint with EndpointSlice (cilium/cilium#42778, @sayboras)
* Reduce memory usage on operator for large AWS environments with many resources (cilium/cilium#42529, @liyihuang)
* Support for Configurable Hubble FlowLog Aggregation (CFP-37472) (cilium/cilium#42011, @mereta)
* Support for FromRequires and ToRequires in Cilium Network Policies has been removed. (cilium/cilium#42615, @TheBeeZee)
* The deprecated REST policy write API has been removed. (cilium/cilium#42705, @squeed)

**Bugfixes:**
* allow missing verbs for cilium-agent cluster role when readSecretsOnlyFromSecretsNamespace is false (cilium/cilium#42790, @kraashen)
* AWS EC2: Fix ENI attachment on multi-network card instances with high-performance networking (EFA) setups (cilium/cilium#42512, @41ks)
* Cilium will now panic when no egress masquerade devices are presented and the IPAM mode is not ClusterPool or Kubernetes, when using IPTables masquerading. This configuration leads to broken IPTables masquerading rules. (cilium/cilium#42531, @ldelossa)
* cilium-cni: Fix bootstrapping agent detection check (cilium/cilium#42584, @pippolo84)
* Cleanup FQDNs that have leaked into the global FQDN cache (cilium/cilium#42485, @sjohnsonpal)
* controller: avoid spurious errors when RemoveControllerAndWait is invoked when the controller does not exist (cilium/cilium#41384, @asdfmi)
* Do not opt-out Endpoint ID 1 from dnsproxy transparent mode. (cilium/cilium#42887, @jrajahalme)
* endpoint/api: wait until endpoints are restored (cilium/cilium#42597, @mhofstetter)
* Ensure cilium-agent gracefully does fallbacks when etcd is in a bad state. (cilium/cilium#42977, @odinuge)
* envoy: pass stream idle timeout from Helm to configmap (cilium/cilium#42499, @mhofstetter)
* Fix a bug that would cause Cilium to not report L4 checksum update errors when the length attribute is missing in ICMP Error messages with TCP inner packets. (cilium/cilium#42426, @yushoyamaguchi)
* Fix a bug that would cause IPsec logs to incorrectly report the XFRM rules being processed as "Ingress" rules. (cilium/cilium#42640, @sjohnsonpal)
* Fix agent local identity leak (cilium/cilium#42662, @odinuge)
* Fix bug that could cause the agent to fail to add XFRM states when IPsec is enabled, thus preventing a proper startup. (cilium/cilium#42666, @pchaigno)
* Fix CiliumNetworkPolicy labels without explicit source field defaulting to empty string instead of "unspec", resolving issues in policy debugging and Hubble flow exports. (cilium/cilium#41168, @pillai-ashwin)
* Fix ipcache issues causing severe issues with the fqdn subsystem (cilium/cilium#42815, @odinuge)
* Fix issue where endpoints got stuck in "waiting-to-regenerate" (cilium/cilium#42856, @odinuge)
* Fix issue where fqdn GC starts too early that results in potentially missed ips in the IPCache (cilium/cilium#42502, @odinuge)
* Fix leak in the policy subsystem (cilium/cilium#42661, @odinuge)
* Fix rare kvstore issue where cilium continues to use an expired lease causing kvstore operations to fail consistently (cilium/cilium#42709, @odinuge)
* Fix trace aggregation for IPv4 Host Firewall, reducing the amount of generated events. (cilium/cilium#42595, @smagnani96)
* In rare cases, the cilium-operator losing the lead of the HA deployment could continue acting as if leading for at most a minute, leading to split-brain problems such as double allocation of pod CIDRs. (cilium/cilium#42920, @bimmlerd)
* KVStoreMesh now correctly respects the CA bundle setting when validating remote cluster certificates (cilium/cilium#42726, @giorio94)
* policy: ensure matched policy order is deterministic to reduce cilium-agent cpu and memory usage (cilium/cilium#42416, @odinuge)
* policy: Fix rare Endpoint Selector Policy Deadlock causing policies to not be updated with new identities (cilium/cilium#42306, @odinuge)
* policy: prevent incorrect mutation of network policy when using policy-default-local-cluster (cilium/cilium#42668, @MrFreezeex)
* redirectpolicy: Avoid recomputing on pod changes that do not change resulting redirect backends (cilium/cilium#42814, @joamaki)
