# Changelog

## v1.19.0-pre.4

Summary of Changes
------------------

**Major Changes:**
* Cilium policy engine now supports enforcing ordered policy rules. (cilium/cilium#42784, @jrajahalme)
* Policy engine adds support for tiers and pass verdicts for ClusterNetworkPolicies (cilium/cilium#42896, @jrajahalme)
* ztunnel: add ZDS server (cilium/cilium#42364, @rgo3)

**Minor Changes:**
* Add the new flags of eni and ipam for cilium agent, and users can use helm to configure the agent directly without using custom CNI (cilium/cilium#43086, @liyihuang)
* bpf:hubble: support policy verdict from L3 devices (cilium/cilium#43015, @smagnani96)
* ces: remove controller maximum limits on QPS (cilium/cilium#43154, @jshr-w)
* Cilium Preflight check no longer includes Envoy Configmaps, making it easier to correctly run. (cilium/cilium#43153, @youngnick)
* clustermesh:  prevent redirecting to endpoints from clusters not exposing certain ports on a port conflict in the MCS-API context (cilium/cilium#43251, @MrFreezeex)
* clustermesh: add trafficDistribution and internalTrafficPolicy to MCS (cilium/cilium#43233, @MrFreezeex)
* clustermesh: MCS now raise a port conflict when exported Service ports are not an exact match (cilium/cilium#43414, @MrFreezeex)
* Correctly handle Gateways with a nil selector when they set NamespacesFromSelector (cilium/cilium#43085, @phuhung273)
* DNS Policies match pattern now support a wildcard prefix(`**.`) to match multilevel subdomain as pattern prefix. For example: `**.cilium.io` now selects both `app.cilium.io`` and `test.app.cilium.io` (cilium/cilium#43420, @fristonio)
* eni: mitigate a race-condition where secondary ENI interface did not get rp_filter disabled (cilium/cilium#43203, @marseel)
* envoy: Allow customize per cluster connections / requests limit (cilium/cilium#43049, @exherb)
* gateway-api: Add retry with exponential backoff for CRD discovery when API server is temporarily unreachable during operator startup (cilium/cilium#43452, @lexfrei)
* hubble: Add --encrypted and --unencrypted flags to filter flows by encryption status (cilium/cilium#43096, @SRodi)
* hubble: add VNI information for Geneve and VXLAN to Hubble flow (cilium/cilium#43398, @glrf)
* k8s: add support for labels specific to cilium-secret namespaces (cilium/cilium#43106, @jusch23)
* multi-pool: add support for excluding IP pools from masquerade (cilium/cilium#43380, @alimehrabikoshki)
* operator: Improve AWS API pagination with adaptive fallback (cilium/cilium#42127, @pillai-ashwin)
* operator: Refactor unmanaged kube-dns pods management into a separate cell. The `unmanaged-pod-watcher-interval` flag now expects a duration value. (cilium/cilium#43344, @HadrienPatte)
* runtime: Add libatomic1 for cilium-envoy dependency (cilium/cilium#43292, @sayboras)
* Some of the secrets generated by Helm use functions that are not idempotent. They are now labelled with `cilium.io/helm-template-non-idempotent: "true"` so that they can easily get identified. (cilium/cilium#43156, @fgiloux)
* ztunnel: add ztunnel as an encryption enum for cilium status (cilium/cilium#43227, @nddq)
