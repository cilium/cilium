# Changelog

## v1.16.0

Summary of Changes
------------------

**Major Changes:**
* Add a readinessProbe to the kvstoremesh container that reports initial synchronization status to support configuring a separate, initial rate-limit to be used while synchronizing. Both clustermesh-apiserver and kvstoremesh now use a high initial rate-limit to decrease start time. (cilium/cilium#30361, @thorn3r)
* Add Kubernetes EndpointSlice synchronization from Cilium clustermesh (cilium/cilium#28440, @MrFreezeex)
* Add support for matching CiliumCIDRGroups in Egress policy rules (cilium/cilium#30624, @chaunceyjiang)
* api: Promote field_mask from experimental to stable, deprecating experimental option (cilium/cilium#30133, @chancez)
* BGP: New BGP APIs can be used to configure Cilium BGP Control Plane. (cilium/cilium#32426, @harsimran-pabla)
* bpf: introduce encrypted overlay datapath support (cilium/cilium#31073, @ldelossa)
* Cilium now supports Kubernetes Service TrafficDistribution. To access this feature, use `--enable-service-topology` when running Cilium. (cilium/cilium#32678, @robscott)
* Cilium now supports the Gateway API GAMMA initiative, allowing configuration of east-west Layer 7 interception using simpler resources. (cilium/cilium#32744, @youngnick)
* cilium: netkit support (cilium/cilium#32429, @borkmann)
* Deploy Envoy as a separate DaemonSet by default rather than running it inside the Cilium Pod (cilium/cilium#30034, @sayboras)
* identity: Allow nodes to be selectable by their labels instead of CIDR and/or remote-node entity. (cilium/cilium#26924, @oblazek)
* Improved performance for DNS lookups (up to 5x reduction in tail latency) when using ToFQDN policies. To avoid drops during upgrades in clusters with ToFQDN policies, it is highly recommended to run Cilium v1.15.6 or newer before upgrading to Cilium v1.16 (cilium/cilium#32769, @gandro)
* iptables: Add rules runtime reconciliation (cilium/cilium#31372, @pippolo84)
* k8s: Add support for Kubernetes 1.30.0 (cilium/cilium#31687, @christarazi)
* KVStoreMesh is now enabled by default in Clustermesh. (cilium/cilium#32912, @marseel)
* loader: attach programs using tcx (cilium/cilium#30103, @rgo3)
* multicast: add CLIs to manage multicast BPF maps (cilium/cilium#31355, @harsimran-pabla)
* NAT source port metrics & table (cilium/cilium#32152, @tommyp1ckles)
* policy/k8s: Add support for CIDRGroupRef in IngressDeny and EgressDeny (cilium/cilium#30933, @pippolo84)
* policy: Add support for port ranges in network policies. (cilium/cilium#32807, @nathanjsweet)
* policy: Add support to watch and read CNP files from directory (cilium/cilium#32599, @tamilmani1989)
* Promote local redirect policy (LRP) feature to stable. (cilium/cilium#33032, @aditighag)
* Support CEL expressions in hubble flow filters (cilium/cilium#31070, @chancez)
* This adds a new policy field, EnableDefaultDeny, which permits the creation of network polices that do not drop non-matching traffic. (cilium/cilium#30572, @squeed)
* This change introduces the BGP control-plane operator. (cilium/cilium#28846, @harsimran-pabla)
